Frontend API Spec
Base URL in React: /api
Local dev proxy: http://localhost:5173/api -> http://localhost:8080/api

Common
- Use fetch with credentials: "include" for APIs that need session login.
- JSON requests use Content-Type: application/json.
- Multipart item create/update requests must not set Content-Type manually.

1. Auth

POST /api/login
Purpose: login and create server session.
Request JSON:
{
  "loginId": "user1",
  "password": "1234"
}
Response JSON:
{
  "memberId": 1,
  "loginId": "user1",
  "nickName": "nickname1"
}

2. Members

POST /api/members
Purpose: signup.
Request JSON:
{
  "loginId": "newuser",
  "name": "Hong",
  "password": "1234",
  "nickname": "newnick",
  "address": {
    "city": "Seoul",
    "street": "Road",
    "zipcode": "00001"
  }
}
Response: created member signup DTO.

GET /api/members/me
Purpose: get logged-in member info.
Requires session.
Response JSON:
{
  "memberId": 1,
  "loginId": "user1",
  "nickName": "nickname1"
}

PATCH /api/members/me
Purpose: update logged-in member info.
Requires session.
Request JSON:
{
  "name": "New Name",
  "password": "5678",
  "nickname": "newnick",
  "address": {
    "city": "Seoul",
    "street": "New Road",
    "zipcode": "00002"
  }
}
Response: updated member DTO.

GET /api/members/{memberId}/shop
Purpose: get public shop info for a member.
Response JSON:
{
  "loginId": "user1",
  "nickName": "nickname1",
  "name": "member1",
  "itemList": []
}

3. Items

GET /api/items?page=0&size=10
Purpose: get item slice for list/infinite scroll.
Response JSON:
{
  "list": [
    {
      "id": 1,
      "name": "item1",
      "description": "description1",
      "price": 1100,
      "status": "SELLING",
      "category": "OUTER",
      "nickName": "nickname1",
      "uploadFileDto": {
        "itemImageId": 1,
        "originalFilename": "item1-image1.jpg",
        "storedFileName": "stored-item1-image1.jpg"
      }
    }
  ],
  "hasNext": true
}

GET /api/items/{itemId}
Purpose: get item detail.
Response JSON:
{
  "id": 1,
  "name": "item1",
  "description": "description1",
  "price": 1100,
  "status": "SELLING",
  "category": "OUTER",
  "nickName": "nickname1",
  "itemImages": [
    {
      "itemImageId": 1,
      "originalFilename": "item1-image1.jpg",
      "storedFileName": "stored-item1-image1.jpg"
    }
  ]
}

POST /api/items
Purpose: create item.
Requires session.
Content-Type: multipart/form-data
Parts:
- itemSaveDto: application/json
- multipartFiles: image files
itemSaveDto JSON:
{
  "name": "New Item",
  "description": "description",
  "price": 10000,
  "category": "OUTER"
}

PATCH /api/items/{itemId}
Purpose: update item.
Requires session and owner permission.
Content-Type: multipart/form-data
Parts:
- itemUpdateDto: application/json
- multipartFiles: optional image files
itemUpdateDto JSON:
{
  "name": "Updated Item",
  "description": "updated",
  "price": 12000,
  "status": "RESERVED",
  "deletedFileIds": [1, 2]
}

DELETE /api/items/{itemId}
Purpose: delete item.
Requires session and owner permission.
Response: 200 OK text response.

4. Images

GET /api/images/{storedFilename}
Purpose: load item image file.
Example:
GET /api/images/stored-item1-image1.jpg
